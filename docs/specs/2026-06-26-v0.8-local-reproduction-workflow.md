# ConsoleDock v0.8 Local Reproduction Workflow

## Goal

`v0.8.0` strengthens the local tester workflow built in `v0.7.0`. A tester should be able to run local Debug Actions, provide small parameter values, add markers, reproduce a problem, and export a report that tells a developer what happened without connecting Xcode.

## Scope

Included:

- Local in-memory Debug Action execution history for the current ConsoleDock session.
- Session-only reuse of the most recent parameter values in the bundled UIKit action parameter form.
- A reproduction timeline in issue reports that combines markers, Debug Action executions, and error/fault logs.
- Issue report sharing as a local `.txt` file item from the bundled UIKit share menu, while keeping copy as plain text.
- Swift and Objective-C samples, unit tests, UI smoke coverage, validators, README, DocC, roadmap, and changelog updates.

Not included:

- Default disk persistence of logs, action history, parameter values, or context.
- Remote commands, uploads, issue creation, or automation platform features.
- Network inspection, crash reporting, automatic route discovery, CocoaPods, XCFramework, or SwiftUI rewrites.

## Design

Debug Action execution history stays in `ConsoleDockDebugActionRegistry` and is reset with registered actions or explicit test cleanup. Each execution records action id, title, group, start time, completion time, outcome, and a compact parameter summary. Handlers still run on the main thread, and history recording must not hold the registry lock while a handler runs.

The bundled UIKit parameter form reads session-only recent values by action id and parameter id. Submitted values are stored only after form validation succeeds and the action is requested. This is intentionally not persisted, so restarting the app or clearing process memory removes the values.

Issue reports add a `Reproduction Timeline` section before full logs. Timeline rows are ordered by timestamp and include marker entries, Debug Action executions, and error/fault log entries. The full `Logs` section remains unchanged so no retained entry is lost.

The share menu continues to offer visible logs, all logs, share issue report, and copy issue report. `Share Issue Report` now provides a temporary local `.txt` item to the system share sheet. ConsoleDock creates this file on demand and does not treat it as persistent storage.

## Testing

Unit tests cover:

- successful, skipped, and failed Debug Action history records.
- parameter summaries are compact and do not create new required parameter behavior.
- history is session-local and clearable.
- issue reports include a timestamp-ordered reproduction timeline.
- issue report file item generation uses `.txt` content without changing the plain text API.

Smoke tests cover:

- parameter form recent values are reused in the Swift sample.
- issue report share/copy actions remain available in Swift and Objective-C samples.
- sample reports include action and marker timeline content through existing public report APIs.
