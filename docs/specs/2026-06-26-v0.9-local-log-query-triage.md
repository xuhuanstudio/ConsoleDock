# ConsoleDock v0.9 Local Log Query And Triage

## Goal

`v0.9.0` makes the bundled local Logs view better for triage when a session contains many entries. A tester should be able to narrow logs with simple structured search terms, jump to the next important visible entry, and share the same local-only reports without connecting Xcode.

## Scope

Included:

- Local advanced query parsing for the bundled Logs search field.
- Structured query tokens for source, level, and entry flags.
- Quoted phrase matching and excluded text terms.
- Compatibility with existing plain-text search behavior.
- Triage-oriented jump actions for next and previous visible error or fault entries.
- Unit tests, UI smoke coverage, README, DocC, roadmap, changelog, and validator updates.

Not included:

- Default persistence of searches, filters, logs, actions, context, or reports.
- Regex, boolean grouping, date ranges, saved searches, or a public query language API.
- Remote upload, remote commands, issue creation, network inspection, crash reporting, automatic route discovery, CocoaPods, or XCFramework distribution.

## Query Design

The search field remains a single local text field. Existing free-text queries continue to work. The query parser recognizes these tokens:

- `source:native`, `source:stdout`, `source:stderr`
- `level:debug`, `level:info`, `level:warning`, `level:warn`, `level:error`, `level:fault`
- `is:partial`, `is:redacted`, `is:truncated`
- quoted phrases such as `"checkout failed"`
- excluded text terms prefixed with `-`, such as `-heartbeat` or `-"cache warmup"`

All recognized query constraints are combined with AND semantics. Text terms match the same searchable entry text used by the current filter: message, source label, and level label. Excluded text terms reject an entry when the searchable text contains that term.

Invalid or unknown `key:value` tokens are treated as ordinary text terms. The parser should be forgiving so testers do not need to learn a strict language before searching.

The existing source and level segmented controls remain available. Query source and level tokens narrow results further; they do not replace the controls or change stored state.

## UI Design

The Logs search placeholder changes from generic search to a compact hint for structured filters. The empty state distinguishes:

- no logs retained;
- logs retained but no results match current filters/query;
- invalid input is not a state because unknown tokens degrade to plain text.

The Jump menu keeps existing latest and first visible error behavior and adds next/previous visible error or fault from the current table selection when possible. These actions operate only on the current visible result set and do not alter filters or stored entries.

## Architecture

Add a private query parser and query model near `ConsoleDockEntryFilter`. Keep it internal to the `ConsoleDock` target unless a real integration need appears later. The parser should be deterministic, allocation-light, and covered by focused unit tests.

`ConsoleDockEntryFilter.filteredEntries` should continue to be the single filtering entry point used by the Logs view. Internally it can parse the query once per reload, then apply:

1. source segmented scope;
2. level segmented scope;
3. query source constraints;
4. query level constraints;
5. query flag constraints;
6. included and excluded text terms.

No changes are needed in `ConsoleDockCore`.

## Testing

Unit tests cover:

- plain text queries remain compatible;
- quoted phrases match exact phrases case-insensitively;
- excluded terms remove matching entries;
- source, level, and flag tokens narrow entries;
- aliases such as `level:warn` work;
- unknown `key:value` tokens behave as text;
- segmented source/level controls combine with query tokens.

UI smoke covers:

- entering at least one structured query in the Swift sample filters logs;
- clearing the query keeps the existing Logs flow working;
- Jump menu still exposes latest and error navigation actions.

Release validation remains the main gate. No new distribution channel or governance gate is added for this release.
