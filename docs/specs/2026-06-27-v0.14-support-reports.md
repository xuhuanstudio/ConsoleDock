# ConsoleDock v0.14 Support Reports

`v0.14.0` adds app-owned Support Reports without changing ConsoleDock's local-only, debug/test-first product boundary.

## Problem

Issue reports and Local Session Archives are useful from the bundled console, but a real app may already have its own feedback, QA, support, or bug-report entry point. That flow may need to attach a ConsoleDock report for a recent time window without exposing the ConsoleDock panel to an end user and without turning ConsoleDock into analytics or telemetry.

## Design

Add a small public Support Report model on the Swift facade:

- `ConsoleDock.SupportReportTimeRange`
- `ConsoleDock.SupportReportOptions`
- `ConsoleDock.SupportReport`
- `ConsoleDock.supportReport(options:)`
- `ConsoleDock.makeTemporarySupportReportFile(options:)`

Add Objective-C/UIKit bridge APIs through `CDKSupportReport` and `CDKConsoleDockUIKit`.

The default report covers the last 10 minutes. Presets cover 5, 10, 30, and 60 minutes. Apps can also request all retained data or an explicit date range. The 60-minute preset is allowed because long manual QA flows are real, but it is not the default and does not change retention.

Reports are generated from currently retained, already-redacted in-memory entries plus current-session Debug Action executions, App Context, diagnostics, and ConsoleDock Health. Evicted entries cannot be recovered. Report text is bounded by `maximumReportCharacterCount` and includes an explicit truncation notice when shortened.

Temporary support-report files are created only on demand. ConsoleDock prunes its own temporary report directory to avoid unbounded accumulation, but the host app still owns upload, sharing, and cleanup for any file URL it keeps.

## Non-Goals

- No analytics or user-statistics collection.
- No background upload.
- No automatic remote issue creation.
- No continuous raw log file.
- No network inspector.
- No crash reporting.
- No remote commands.
- No CocoaPods or XCFramework work.

## Validation

- Unit tests cover last-minute filtering, the 60-minute preset, reversed date ranges, report truncation, temporary file output, temporary file pruning, and Objective-C/UIKit bridge APIs.
- Swift API surface validation locks the new public Swift and UIKit-visible API.
- README, simplified Chinese README, DocC, privacy docs, roadmap, sample walkthrough, and changelog describe Support Reports as on-demand local reports rather than telemetry.
